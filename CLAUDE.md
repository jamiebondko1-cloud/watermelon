# 이 프로젝트에 대해

- **목적**: 개발 연습용 폴더입니다. (실제 서비스가 아니라 배우고 실험하는 공간)
- **기술 스택**
  - **Next.js** — 웹사이트(화면)를 만드는 도구
  - **Supabase** — 회원정보·데이터를 저장하는 온라인 데이터베이스(창고)
  - **Vercel** — 만든 웹사이트를 인터넷에 띄워주는 서비스(배포)
- **배포 방식**: 코드를 **GitHub에 올리면(push)**, 이미 연결된 **Vercel이 자동으로 인터넷에 배포**합니다. 별도 배포 버튼을 누를 필요가 없습니다.

# 함께 일하는 방식 (중요 — 항상 지킬 것)

이 폴더의 사용자는 **비개발자**입니다. 아래 원칙을 반드시 따르세요.

## 1. 쉬운 말로 설명하기
- 개발 용어가 나오면 **비개발자도 이해할 수 있게 풀어서** 설명합니다.
  - 예) "환경변수" → "비밀번호 같은 설정값", "커밋" → "작업 내용 저장", "배포" → "인터넷에 올리기"
- 전문 용어를 꼭 써야 하면, 옆에 괄호로 쉬운 뜻을 같이 적습니다.

## 2. 지금 뭘 하는지 자주 말하기
- 작업을 시작할 때, 진행 중일 때, 끝났을 때 **"지금 무엇을 하고 있는지" 자주 알려줍니다.**
- 한참 말없이 작업하지 말고, 단계마다 짧게 현재 상황을 보고합니다.

## 3. 시킨 것만, 단계적으로
- 사용자가 **지시한 것만** 수행합니다. 시키지 않은 기능을 미리 만들거나 멋대로 확장하지 않습니다.
- 한 번에 여러 가지를 몰아서 하지 않고, **한 단계씩** 진행합니다.

## 4. 할 수 있는 건 최대한 대신 하기
- 코드 작성, 파일 수정, 명령어 실행 등 **AI가 할 수 있는 일은 최대한 직접 처리**합니다.
- **꼭 사람이 직접 해야 하는 일**(예: Supabase/Vercel 사이트 로그인, 결제, 비밀번호 입력, 마우스로 버튼 클릭해야 하는 설정)만 골라서, **무엇을·어디서·어떻게 하는지 순서대로 쉽게 안내**합니다.

## 5. 끝나면 바로 확인할 수 있게 열어주기
- 개발(작업)을 마치면, 사용자가 **결과를 즉시 눈으로 확인**할 수 있도록 합니다.
  - 화면 작업이면 미리보기(개발 서버)를 띄우거나 브라우저로 해당 페이지를 열어줍니다.
  - 배포 결과면 배포된 주소(링크)를 안내합니다.
- "직접 확인해 보세요"라고만 하지 말고, **바로 볼 수 있는 상태로 만들어 줍니다.**
